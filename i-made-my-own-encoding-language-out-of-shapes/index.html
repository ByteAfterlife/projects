I made my own encoding language.. out of shapes

Byte December 6, 2025

Firstly, I'll just say, I get lots of ideas that I then later discover were thought of like 2 decades ago by someone else, so if this is another one of those times and someone already made this, then sorry, I didn't research it or anything, I just made this for fun. Also this went through many revisions because I got more ideas for what to add and probably will get more, if they do, I'll make sure to post it here!

So I was bored one day and came up with a encoding language.. There's 2 rows. One is the "grouping" row, the next is the "index" row as I call it, you group the alphabet letters into groups of 3, so group 1 is "ABC", group 2 is "DEF", 3 is "GHI", and group 9 is "YZ" now, originally I didnt plan to use shapes, I planned to use dots. Anyway how the rows used to work is the first row, the "grouping" row is for which group of letters your letter is in, let's say I want to write the message "Hi" what we'd do is since the letter "H" is in the "GHI" group, which is 3, we put 3 dots in the first column, first row, to represent "GHI." by the way you can make these rows with just some simple lines, nothing complicated like a special algorithm for the columns. Now, in the second row, first column, we'd put two dots, why? well H is 2nd in the "GHI" group just like that, we wrote H in my silly language, for I, 2nd column first row would be 3 dots again for "GHI", then 3 dots 2nd row 2nd column for I. And we just wrote "HI" (haven't invented capitalisation yet!) Also I didn't really make a requirement for how the dots have to be placed, you can put them in a straight line, a triangle for 3, square for 4, whatever, as long as its within the row and row and column. Here's what it would look like saying "HI"

Also I made sure to place the dots randomly to show THE DOT PLACEMENT DOESNT MATTER as long as its contained in your cell (row and column). Now, this was V1 of my secret language, so once when I arranged the dots like a triangle, I realized "Wait why don't we just use shapes?" So I decided to use shapes. Also you may use a dot for 1, so for "A" you would put a dot in column 1 row 1, and another in column 1 row 2. Now "HI" looked like this:

Also the orientation doesn't matter of the shape, you can make horizontal or vertical lines, upside down triangles, sideways triangles, squares at a 45 degree angle, anything, as long as the vertices match what you want. Simply just count the vertices and that's the dot, a triangle has 3 vertices, so that equals "3" or the 3rd group, "GHI" and a line has 2, the 2nd letter in the GHI group, just like that, we got "HI" but out of shapes. Then in class when I was experimenting with it, I realized.. "Wait what if I just flipped it upside down" Since the orientation of the shapes doesn't matter to me, you can make a upside down triangle, but now what if you got it upside down (the H and I on the top is just for guidance, if using for real, which I'd be surprised if you did, you wouldn't write the decoded letter above it) So what if you got it upside down, then it would look like this?

Now it says "IF".. HUH? NO! I wanted to say Hi! NOT IF. So what did I do? make an orientation symbol. The orientation symbol is an "O" with a line under it. and it always goes on top of the code. So Hi would look like this now with V3 (my orientation symbol update lol)

You just gotta follow the global rule of "O symbol is always above" now if i turn it any direction, the O symbol wouldn't be on top, which would signal to the reader to fix the orientation. Also, once when I wanted to write "S" I didn't feel like drawing a shape with 7 sides, so you can also put multiple shapes into a cell as long as the vertices add up to what you wanted originally, so a square and a triangle's vertice count adds up to 7, which would be the "STU" group. then draw a simple dot for the 1st letter in "STU", "S" and you got S. Also the 2nd row should always have less than 4 vertices. Since letters are grouped in 3, and a group cannot have more than 3, if you see a square in the second row ( with correct orientation, of course) then it's a mistake since a square has 4 vertices

I also made another edition called "Additive", which instead of using groups and indexes of letters in those groups, uses the closest group to the letter, and the second row is addition to get to that letter. So D would be a single dot in the top row, which would be.. "ABC"? What??? Yes, in my first edition, that wouldn't make sense. But in this edition, the second row would be one more dot, which would signal "what's 1 letter after C?" which is D. so D would look like this in my Additive edition:

The first row always equals the last letter of the group, so if you drew a triangle, which would mean the group "GHI", "I" would be the letter, then the second dot would be what you want to add, so if you want to write "K", you would make a triangle in the top row for "GHI", then the second row would have a line or any other two vertices shape, which would mean "I" + 2, which would be "K"

This is a really silly idea I had, if you want to try using it, have fun! Much thanks for reading. Also I know this isn't my usual tech stuff but I found it a fun post to make for those interested in encoding and stuff.
