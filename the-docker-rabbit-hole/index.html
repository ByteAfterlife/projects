The docker rabbit-hole

Byte November 29, 2025

I've been using docker for a while now, docker compose, I have at least 20 images. Docker is a VERY awesome software for managing servers. I used it to setup my Keycloak systems, I even once made my own docker image that made an API for an app that converted a plaintext token to a JWT because it wouldn't work without one. Making docker images involves a Dockerfile, which is in my opinion its own programming language. Anyway, once I started with docker, I can't leave. I ran a Mumble server with it, Keycloak as mentioned before, local image converter, text decoder and decryption, nextcloud, wireguard, LDAP servers, and even a drawing app (I love you excalidraw)

In summary what docker does is basically you run server apps in containers which locks them to only be able to use the files and computer resources and ports you allow it, you're basically locking down a server application into a certain environment. There's also a thing called docker compose which basically lets you make these apps from a file, like a instruction file automating the setup!
