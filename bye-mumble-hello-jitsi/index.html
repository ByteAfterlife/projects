Bye Mumble, Hello Jitsi

Byte December 6, 2025

So, as I discussed in my Mumble setup post, I made my own mumble server and port forwarded to make it accessible to the world! However, my family didn't like the limits: No video, no screensharing. Now, I've known about Jitsi Meet for a while. Not the cloud hosted one they provide, but the self hosted one. Jitsi Meet is a self hostable server app that lets you have meetings with people (screenshare and video and much more included) Now I'm gonna be completely transparent and say I've failed lots of times setting up an actually working Jitsi server. But this time it was surprisingly easy, I downloaded their pre-made docker compose file, it worked surprisingly! I was expecting 3 hours of debugging to just get the welcome screen, let alone a working meeting. But surprisingly, it worked first try, I got into a meeting, and everything was working. Now, I needed some form of authentication, if this was internally running on my LAN network, it wouldn't matter too much, but if I'm gonna port forward this and make it publicly accessible, I need some sort of security. Then I found myself configuring it to use an LDAP backend, and I added to the docker compose file an LLDAP (Light LDAP) (not LDAP) server. LLDAP is the simplest implementation of an LDAP server I know. It requires minimal setup, and definitely doesn't require 1 hour of commands to add an attribute. It even includes a web interface to manage users. Awesome! Now I was originally considering using Jitsi's internal user system for authentication, because Jitsi's user auth manager, prosody, lets you just internally create users with passwords in it's database and you can use that. But I like a clean web UI, so yes, I deployed a LDAP server just so I get a pretty web interface for managing users. I don't care that much, this isn't a enterprise grade server for millions of concurrent clients, I'm running these off my laptop for ~4 people, so I don't mind too much on resource usage. So just like that, my stack is done! Jitsi meet + LDAP for user management was epic. Then I setup the port forwarding and stuff, and just like that, we did it! Here's some screenshots.

UPDATE: Ok so something was very wrong, any user that joins gets moderator rights... so what I had to do is enable guests, then take a completely different approach. Instead of everyone in my family having an account on LDAP, I just have one, since I guess Jitsi just gives all LDAP users moderator in meetings. But now that guests are enabled, anyone can join the meeting. The only time they cant is when a moderator isn't in the meeting, but if a moderator is in the meeting, they can just join, now I could enable the lobby, which would 100% solve the problem, but what if I forget? Well I just discovered a community made plugin called "lobby_autostart_on_owner" that automatically enables the lobby after a moderator joins to prevent users from joining without a moderator's permission. So the new flow: Only I have an account, the lobby is automatically enabled whenever I join so I don't have to remember to myself. Fine solution for now I guess, I also removed LLDAP completely since all that just for one user account is a bit too much, so now its just internally managed via that prosody internal database I mentioned earlier.
